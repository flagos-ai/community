## Summary

**(Required)** This FEP covers the features and improvements delivered in TransformerEngine-FL between commits `1f985114` and `badccf26`, including:

1. **Upstream synchronization with TE Release V2.14** — integrates NVIDIA TransformerEngine upstream v2.14 (304 commits, v2.9.0 → v2.14.0) while preserving the custom plugin system.
2. **New vendor backend: KunlunXin** — adds KunlunXin chip vendor operator support (flash attention, register ops).
3. **New vendor backend: ENFLAME** — adds ENFLAME chip vendor operator support (flash attention, register ops).
4. **FlagOS grouped GEMM operator** — implements `te_general_grouped_gemm` op for the flagos backend based on FlagGems, supporting forward and backward computation.
5. **CI/CD enhancements** — MetaX MACA workflow support, coverage reporting to FlagCICD platform, workflow refactoring with integration tests, and switch to FlagCICD MetaX runner.

Repository: [TransformerEngine-FL](https://github.com/flagos-ai/TransformerEngine-FL)

## Motivation

FlagOS 2.1 requires TransformerEngine-FL to broaden hardware coverage for domestic chip ecosystems (KunlunXin, ENFLAME), keep pace with upstream NVIDIA TE enhancements (FP8/MXFP8, Blackwell support, FSDP2), and provide a pure-Triton grouped GEMM operator for the flagos backend to reduce vendor dependency. The CI/CD improvements ensure continuous validation across CUDA, MetaX, and other platforms via the FlagCICD infrastructure.

### Goals

**(Required)**

- Synchronize with NVIDIA TransformerEngine upstream release v2.14, incorporating quantization (MXFP8, NVFP4), Blackwell architecture support, FSDP2, and new fused operators.
- Preserve and adapt the FlagOS plugin system (plugin OP API signatures, `te_device_type()` multi-backend compatibility, symbol renames).
- Add KunlunXin as a vendor backend with flash attention and operator registration.
- Add ENFLAME as a vendor backend with flash attention and operator registration.
- Implement `te_general_grouped_gemm` for the flagos backend using FlagGems, supporting both forward and backward passes.
- Establish MetaX MACA CI workflow with unit tests, integration tests, and coverage reporting on FlagCICD platform.

### Non-Goals

- Fused attention backend for flagos/reference backends (not yet supported, tracked separately).
- NVFP4/MXFP8 quantization on non-CUDA platforms (upstream-only feature for now).
- Inference optimization for TE-FL (out of scope for this cycle).

## Proposal

### 1. Upstream Synchronization with TE V2.14

Integrate NVIDIA TransformerEngine upstream release v2.14 via tree replacement strategy:

**Upstream Enhancements Incorporated:**
- **Quantization & Precision:** MXFP8 grouped GEMM, NVFP4 grouped quantization with Hadamard transform, QuantizedTensor in FusedAdam optimizer.
- **Architecture Support:** Blackwell (sm120) fused attention with cuDNN 9.18.1+, deterministic training on Blackwell, grouped GEMM cuBLAS bindings.
- **Distributed Training:** FSDP2 with DTensor-aware optimizer states, collective GEMM with FP8/MXFP8 for JAX, GroupedTensor torch ops.
- **Operators:** Fused RMSNorm dLN with add-through, MoE grouped MLP ops, configurable philox rounds.

**FlagOS Plugin System Preservation:**
- Synced plugin OP API signatures with upstream csrc changes (fused_attn_fwd/bwd parameters, attention backend dispatch).
- Patched new upstream CUDA hardcoding to `te_device_type()` for multi-backend compatibility.
- Fixed stale references to renamed upstream symbols (e.g., `CPUOffloadEnabled` → `is_cpu_offload_enabled()`).

Related PR: #62.

### 2. KunlunXin Vendor Backend

Add KunlunXin vendor backend at `transformer_engine/plugin/core/backends/vendor/kunlunxin/`:
- `kunlunxin.py` — device management and platform utilities.
- `flash_attention.py` — flash attention operator implementation.
- `register_ops.py` — operator registration for the plugin dispatch system.

Related PR: #66.

### 3. ENFLAME Vendor Backend

Add ENFLAME vendor backend at `transformer_engine/plugin/core/backends/vendor/enflame/`:
- `enflame.py` — device management and platform utilities.
- `flash_attention.py` — flash attention operator implementation.
- `register_ops.py` — operator registration for the plugin dispatch system.

Related PR: #61.

### 4. FlagOS Grouped GEMM Operator

Implement `te_general_grouped_gemm` op for the flagos backend:
- Based on FlagGems Triton kernels.
- Supports both forward and backward computation, distinguished by `grad` parameter.
- Enables grouped GEMM without vendor-specific CUDA dependencies.

Related PR: #55.

### 5. CI/CD Enhancements

- **MetaX MACA workflow** (#48): Add CI workflow for MetaX MACA platform with unit tests.
- **Coverage reporting** (#58): Upload unittest coverage report to FlagCICD platform; access FlagCICD runner.
- **Workflow refactoring** (#60): Refactor CI workflows, add integration tests, switch to FlagCICD MetaX runner.

Related PRs: #48, #58, #60.
