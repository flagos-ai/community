# Contributor Guide

Welcome to FlagOS! Choose your entry point based on your goal and role:

## I want to…

| Scenario | Entry Point |
|----------|-------------|
| Open a PR to fix a bug or add a feature | [How to Contribute Code](CONTRIBUTING.md) |
| Understand how the FlagOS community operates | [GOVERNANCE.md](../GOVERNANCE.md) |
| Understand how decisions are made (lazy consensus, voting) | [Decision-Making Guide](decision-guide.md) |
| Understand how meetings are run (TSC/SIG meeting norms) | [Meeting Operations Guide](meeting-guide.md) |
| Learn about roles and promotion paths | [Role Definitions & Promotion Path](roles.md) |
| Understand how TSC elections work | [TSC Election Process](election.md) |
| Learn how to write a SIG annual report | [SIG Annual Report Guide](sig-annual-report.md) |
| Understand how communication channels are managed | [Communication Channel Operations Guide](communication-guidelines.md) |
| Review PRs or understand review standards | [Code Review Guide](review-guide.md) |
| Perform Issue Triage | [Issue Triage Guide](issue-triage.md) |
| Set up a development environment | [Dev Environment Setup](dev-setup.md) |
| Propose a FEP (design proposal) | [FEP Writing Guide](fep-guide.md) |
| I'm a chip vendor and want to onboard with FlagOS | [Chip Vendor Onboarding Guide](chip-vendor-guide.md) |
| Learn the documentation standards | [Documentation Style Guide](doc-style-guide.md) |
| Run into a problem | [FAQ](faq.md) |
