# Incubating Working Groups (WGs)

A WG is the precursor to a SIG. When a technical area does not yet have enough contributors to sustain a formal SIG, it incubates as a WG.

## WG vs. SIG

| Dimension | WG | SIG |
|-----------|-----|-----|
| Chair Requirement | 1 Organizer is sufficient | >=1 Chair (target >=2) |
| Decision Authority | Cannot approve FEPs | Can approve FEPs within SIG scope |
| Meeting Requirement | Recommended but not mandatory | Biweekly meetings (mandatory) |
| OWNERS Authority | No binding approve | Approvers can merge code |

## Promotion Criteria

A WG may apply for promotion to SIG when the following conditions are met:
1. >=3 active Contributors (with PRs merged in the past 3 months)
2. >=1 demonstrable deliverable (code, paper, demo)
3. At least 1 Chair candidate
4. TSC 2/3 vote approval
