# sig-documentation: Documentation Infrastructure (Planned)

> Documentation is the number-one infrastructure for user growth.

## Reason for Deferred Activation

Documentation is critically important, but a standalone SIG is not needed during the launch phase. A virtual documentation team can function effectively by having the TSC designate 1 documentation lead, with each active SIG assigning 1 documentation liaison.

## Planned Responsibilities

- Establish documentation standards (minimum README requirements per module)
- Maintain docs.flagos.io (or flagos-wiki)
- Author Getting Started guides, Tutorials, and API Reference
- Maintain bilingual content (Chinese and English)
- Publish a Changelog for each release

## Current Interim Arrangement

The TSC Community Manager serves as the documentation lead, with each SIG assigning a documentation liaison to form a virtual team.
