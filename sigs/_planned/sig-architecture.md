# sig-architecture: Cross-Module Architecture (Planned)

## Reason for Deferred Activation

TSC members can directly assume architecture review responsibilities; a standalone SIG is not needed. Cross-module FEPs are approved directly by the TSC.

## Planned Responsibilities

- Cross-module architecture decisions
- Process change proposals (governance, CI, release process, etc.)
- Technical debt management

## Current Interim Arrangement

The TSC itself serves as sig-architecture.
