# sig-release: Release Engineering (Planned)

## Reason for Deferred Activation

The current release frequency and complexity do not warrant a standing SIG. Appointing a Release Manager per release cycle is sufficient.

## Planned Responsibilities

- Maintain the release calendar
- Branch and tag management (manage-release.py)
- Author Release Notes and release announcements
- Maintain release-*.yaml and chip-targets-*.toml

## Current Interim Arrangement

The TSC directly manages the release cycle, appointing a Release Manager on a per-release basis.
